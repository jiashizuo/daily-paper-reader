# 论文《Rule-Guided Reinforcement Learning Policy Evaluation and Improvement》总结

## 1. 核心问题与整体含义（研究动机和背景）
- **研究动机**：深度强化学习（DRL）在游戏等领域取得巨大成功，但其决策缺乏可解释性，且难以被信任；实际应用中，DRL往往需要结合领域知识才能表现更好。因此，如何利用领域知识来评估和改进深度强化学习策略是一个重要挑战。
- **核心问题**：如何通过规则的形式化表示和泛化，来发现策略中的弱点（即未学会泛化的决策），并据此提升策略性能，同时提供可解释的说明。
- **整体含义**：本文提出一种端到端的规则引导框架 **LEGIBLE**，将策略决策转化为可理解的符号规则，利用领域知识（变体关系MRs）泛化规则，并通过规则引导执行来评估和改进策略。该方法不仅提升了累积奖励，还提供了弱点的解释。

## 2. 方法论
### 核心思想
- 从训练好的深度RL策略中挖掘二值规则（正规则：采取某动作；负规则：避免某动作），构成策略的部分符号表示。
- 利用**变体关系（Metamorphic Relations, MRs）**，将领域知识（如对称性、等价关系）表达为特征-动作之间的对应关系，从而泛化规则到相关但不同的状态。
- 通过规则引导执行（强制应用正规则、阻止负规则）与原策略进行对比，若规则引导后累积奖励显著提升，则表明该规则对应的泛化策略存在弱点（即策略未学到该泛化），同时规则本身提供了解释。
- 最后，组合所有能提升性能的泛化规则集，形成改进后的执行策略。

### 关键技术细节
1. **规则挖掘**：使用LIME计算特征重要性，结合RIPPER算法为每个动作和极性（正/负）分别挖掘规则，规则形式为 `⊗action(a) ← ∧ fi = vi`。优化目标为FOIL信息增益与特征重要性的乘积。
2. **规则泛化**：定义特征关系 `fr = ⟨a, a', i, j, R⟩`，表示动作a与a'、特征i与j之间存在关系R（如相等、旋转等）。MR是fr的集合。通过MR，自动从原始规则生成泛化规则集`Mgen(ρ)`，固定点计算。
3. **规则引导执行**（Algorithm 1）：每步检查触发规则，如果只有一条正规则触发则强制执行；否则将负规则对应的动作Q值设为负无穷，然后选取最高Q值动作。支持组合正负规则。
4. **策略改进**（Algorithm 2）：贪心地选择所有能检测到弱点的泛化规则集，组合后引导执行，生成改进策略。

### 公式与算法流程（文字描述）
- 规则形式：`+action(a) ← ...` 或 `-action(a) ← ...`
- 泛化规则集生成：`Mgen(ρ) = {ρ} ∪ ∪_{mr∈M} {Mgen(ρ') | ρ mr ρ'}`
- 规则引导执行伪代码：见Algorithm 1（循环n个episode，每步检查规则触发、处理冲突、选择动作）。
- 贪心选择组合：Algorithm 2（遍历所有规则，若其泛化规则集引导后奖励显著高于基线，则加入组合；重新评估组合；保持最大奖励）。

## 3. 实验设计
### 使用的环境/场景
- **PAC-Man**（6个关卡）：分别有/无胶囊版本（small, medium, original）以及对应的无胶囊版本（-nc）。状态维度分别为69和117。
- **highway-env**（5个环境）：highway, highway-fast, merge, intersection, roundabout。状态为相对位姿的7个属性×10辆车。

### Benchmark与对比方法
- **基线方法**：
  - **无引导执行**（原始策略π）
  - **随机测试 (Random Testing, RT)**：随机选择~3%的状态，强制或阻断随机动作。
  - **随机规则 (Random Rules, RR)**：生成与MR规则集大小相近、但随机组合特征和极性的规则集，作为MR的消融对照。
- **评价指标**：平均累积奖励及其标准误；检测到的弱点比例（通过Welch检验，p<0.05确定显著提升的规则集）。

### 训练细节
- 使用stable-baselines3训练DQN，PAC-Man训练2.5M或5M步，highway-env训练500k步。
- 每个实验重复5次（不同随机种子训练出的策略）。

## 6. 主要结论与发现
- **弱点检测**：规则引导执行（MT）在所有环境均能检测到弱点（平均5%~24%的规则集显著提升性能），显著高于随机测试（RT）的0%~9%。RR在某些场景高于MT但在多数场景低于MT，表明领域知识（MR）是有效的。
- **策略改进**：贪心组合所有能检测弱点的泛化规则集后，引导执行在11个环境中的10个获得显著奖励提升（最高达273%），且远超延长训练的效果（延长训练几乎无改善或提升很小）。
- **解释性**：规则本身提供了弱点解释，例如“不要向北走，如果北边有墙且无食物”。

## 7. 优点
- **新颖性**：首次将变体测试（MT）引入RL，为领域知识表达提供了一种形式化框架。
- **可解释性**：规则作为符号表示，既可用于评估弱点，又能提供人类可理解的解释。
- **有效性**：在多种环境中一致提升了累积奖励，且提升幅度大，表明该方法能有效弥补策略泛化不足。
