## Abstract
In this work, we present the first study to explore inference-time scaling on table reasoning tasks. We develop and evaluate two post-training strategies to enable inference-time scaling: distillation from frontier model reasoning traces and reinforcement learning with verifiable rewards (RLVR). For distillation, we introduce a large-scale dataset of reasoning traces generated by DeepSeek-R1, which we use to fine-tune LLMs into the Table-R1-SFT model. For RLVR, we propose task-specific verifiable reward functions and apply the GRPO algorithm to obtain the Table-R1-Zero model. We evaluate our Table-R1-series models across diverse table reasoning tasks, including short-form QA, fact verification, and free-form QA. Notably, the Table-R1-Zero model matches or exceeds the performance of GPT-4.1 and DeepSeek-R1, while using only a 7B-parameter LLM. It also demonstrates strong generalization to out-of-domain datasets. Extensive ablation and qualitative analyses reveal the benefits of instruction tuning, model architecture choices, and cross-task generalization, as well as emergence of essential table reasoning skills during RL training.

- **研究动机**：此前推理时扩展（inference-time scaling）在数学推理、多模态、机器翻译等领域取得了显著效果（如 OpenAI o1、DeepSeek-R1 等），但在表格推理任务中尚未被系统探索。表格推理具有独特的挑战：需要理解多样化的单元格内容、跨列对齐数据、执行多步推理（包括聚合和数值运算），且表格输入往往很长且结构密集。
- **整体含义**：本文首次将推理时扩展应用于表格推理任务，旨在提升大语言模型（LLM）在表格 QA、事实验证、自由问答等场景下的推理能力，并探索两种主流后训练策略——蒸馏（distillation）和基于可验证奖励的强化学习（RLVR）——的有效性。

## 2. 方法论

### 核心思想
- 通过后训练使 LLM 在推理时生成更长的、结构化的推理链（chain-of-thought），即“先思考后回答”，从而提升表格推理准确性。
- 提出两种模型变体：
  - **Table-R1-SFT**：基于 DeepSeek-R1 生成的推理轨迹进行监督微调。
  - **Table-R1-Zero**：使用基于可验证奖励的强化学习（RLVR）训练，采用 GRPO 算法。

### 关键技术细节
- **训练数据收集**：整合三类任务的数据集：短形式表格 QA（WTQ、HiTab）、表格事实验证（TabFact）、自由形式表格 QA（FeTaQA）。总计 48,563 样本用于 RLVR，33,601 样本用于 SFT。
- **蒸馏方法**：使用 DeepSeek-R1 为每个训练样本生成长 CoT 推理轨迹（包括逐步推理和最终答案），然后通过自动评估器筛选出答案正确的样本。
- **RLVR 方法**：
  - 算法：Group Relative Policy Optimization (GRPO) 结合 DAPO 改进（token 级损失计算、非对称裁剪、省略 KL 惩罚项）。
  - 奖励设计：包括**准确率奖励**（短 QA 用精确匹配，事实验证用分类准确率，自由问答用 BLEU+ROUGE-L 均值）和**格式奖励**（鼓励输出 `<think>...</think><answer>...</answer>` 的 JSON 格式）。
- **训练细节**：基础模型为 Qwen2.5-7B-Instruct 和 Llama-3.1-8B-Instruct。SFT 训练 3 个 epoch，RL 训练 2 个 epoch，学习率分别为 1e-5 和 1e-6，最大序列长度 20480 / 4096 + 1024 等。

## 3. 实验设计

### 数据集与基准

- **评估基准**：涵盖 13 个数据集，分为：
  - **域内**：FeTaQA、TabFact、WTQ、HiTab
  - **域外**：TQA 类（TabMCQ、TabMWP、FinQA）、TFV 类（InfoTabs、PubHealthTab、Feverous）、FF-TQA 类（ToTTo、QTSumm、RotoWire）
- **评价指标**：TQA 使用精确匹配（EM，并用 GPT-4.1 mini 二次评估非匹配项）、TFV 使用分类准确率、FF-TQA 使用 BLEU 和 ROUGE-L。

### 对比方法

## 6. 主要结论与发现

- **推理时扩展对表格推理有效**：Table-R1-Zero 和 Table-R1-SFT 在域内任务上显著提升，且在 7B 规模匹配甚至超越 GPT-4.1 和 DeepSeek-R1。
- **RLVR 方法优于蒸馏**：Table-R1-Zero 在域外泛化上更优，且能自主发现多步推理和表格特定技能（列关联、语义理解、算术计算）。
- **指令调优至关重要**：instruct 模型在训练稳定性和最终性能上均优于 base 模型。
- **模型架构影响**：Llama-3.1-8B 在域内性能更强，但 Qwen2.5-7B 域外泛化更好。
- **格式奖励对迁移能力重要**：去除格式奖励后，域外性能下降，尤其是短 QA 和自由形式 QA。
- **跨任务泛化存在差异**：TQA 训练可提升 TFV，但对 FF-TQA 帮助有限。
